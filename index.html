Software Requirements Specification (SRS)

Aplikasi Mobile Pencatat Ide Proyek / Penelitian Mahasiswa

1. Pendahuluan

Dokumen ini menjelaskan spesifikasi kebutuhan perangkat lunak untuk aplikasi mobile yang bertujuan membantu mahasiswa mencatat dan mengelola ide proyek atau penelitian secara efisien.

2. Tujuan

Tujuan dari sistem ini adalah menyediakan platform bagi mahasiswa untuk mencatat, menyusun, dan mengelola ide-ide proyek atau penelitian yang dapat diakses secara mobile dan tersinkronisasi.

3. Lingkup Aplikasi

Aplikasi akan memiliki fitur pencatatan ide, pengelompokan berdasarkan kategori atau status, penambahan referensi atau lampiran, sinkronisasi cloud, serta pengingat dan riwayat perubahan.

4. Deskripsi Umum

  • Pengguna utama: Mahasiswa
  • Platform: Android dan iOS
  • Teknologi: Flutter, Firebase (untuk autentikasi dan sinkronisasi data)

5. Fitur Utama

  • Mencatat ide proyek atau penelitian
  • Membuat kategori atau tag untuk ide
  • Menambahkan referensi atau tautan
  • Upload file (gambar, PDF, dll)
  • Sinkronisasi dengan cloud
  • Riwayat perubahan/catatan versi
  • Pengingat jadwal atau tenggat waktu

6. Kebutuhan Fungsional

  • Pengguna dapat membuat, mengedit, dan menghapus ide
  • Pengguna dapat login dan logout
  • Pengguna dapat menyaring dan mencari ide berdasarkan kategori atau kata kunci
  • Pengguna dapat mengatur pengingat untuk ide tertentu

7. Kebutuhan Non-Fungsional

  • Antarmuka pengguna yang responsif dan mudah digunakan
  • Keamanan data pengguna melalui autentikasi dan enkripsi
  • Performa cepat, bahkan dengan banyak ide tersimpan
  • Ketersediaan layanan cloud minimal 99%

8. Batasan

Aplikasi membutuhkan koneksi internet untuk sinkronisasi data. Fitur unggah file dibatasi ukuran maksimal 10MB.

9. Asumsi dan Ketergantungan

  • Pengguna memiliki akun mahasiswa aktif
  • Perangkat mobile mendukung sistem operasi Android 8+ atau iOS 13+
  • Koneksi internet stabil tersedia saat sinkronisasi
